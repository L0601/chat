# LunaDesk UI 与多 API 配置说明

## 设计基线

- 聊天内容优先，顶栏和输入区保持轻量。
- 使用中性浅色/深色语义色，不依赖装饰性渐变表达层级。
- 触控区域不小于 48dp；图标均提供无障碍描述。
- 助手回复使用开放式排版，用户消息使用紧凑气泡。
- 复制按钮叠放在消息右上角，不创建额外操作行。

## API 配置

每套配置独立保存：

- 配置名称
- OpenAI 兼容 API 地址
- API Key
- 模型 ID
- 温度
- 最大输出 Token

配置以 JSON 形式保存在 DataStore 的 `api_profiles_v1` 中。旧版单配置字段不迁移；首次启动会生成一套空白“默认配置”。API Key 根据当前产品选择明文存储，界面默认遮挡，但仍可能随设备备份被导出。

地址可以填写 `https://host` 或 `https://host/v1`，网络层会统一构造 `/v1/models` 和 `/v1/chat/completions`，不会重复添加 `/v1`。

## 切换规则

- 侧边栏和设置页均可切换当前配置。
- 切换配置保留当前聊天上下文。
- 正在生成回复时禁止切换，必须先停止生成。
- 编辑采用显式保存；离开未保存页面前需要确认。
- 模型既可以手动填写，也可以从服务端列表中选择。

## 流式滚动

- 新一轮生成默认跟随最新内容。
- 用户在生成期间开始拖动列表后，本轮永久暂停自动跟随。
- 手动滚到底部或点击“查看最新”不会恢复本轮跟随。
- 下一次发送消息时自动恢复。
- 自动滚动使用 250ms 合并动画；高频 SSE 更新不会堆积多个动画。
- 流式阶段使用稳定纯文本，生成完成后再渲染 Markdown 和公式。
